### 1. 增加"不适用"状态

**问题**：有些检查项因文件没有相关内容而显示"通过"，不够准确。

**解决方案**：
- 在 `CheckStatus` 枚举中添加 `NOT_APPLICABLE` 状态
- 当文档中无相关内容时，检查器返回"不适用"状态

**实现示例**：
```python
class FigureCaptionChecker(BaseChecker):
    def check(self, document_processor):
        # 查找图标题
        figure_captions = [...]
        
        # 当文档中无图时，返回"不适用"
        if not figure_captions:
            return self.create_not_applicable_result("文档中未发现图标题")
        
        # 有图时，正常检查...
```

**效果**：
- ✅ 通过：文档有图且格式正确
- ❌ 未通过：文档有图但格式错误
- ○ 不适用：文档中没有图

**影响的检查项**：
- 图标题格式检查（无图时）
- 表标题格式检查（无表时）
- 公式编号检查（无公式时）
- 续表检查（无续表时）
- 其他依赖特定内容的检查项

### 2. 显示期望值和实际值

**问题**：检查结果只说"不符合要求"，没有显示具体数值。

**解决方案**：
- 在 `CheckIssue` 中添加 `expected_value` 和 `actual_value` 字段
- 检查器在发现问题时记录具体数值

**实现示例**：
```python
class TextFontChecker(BaseChecker):
    def check(self, document_processor):
        expected_font = '宋体'
        expected_size = 12
        actual_font = '黑体'
        actual_size = 14
        
        # 创建问题时，添加期望值和实际值
        issues.append(self.create_issue(
            position=f"第{page}页 第{paragraph}段",
            description="字体不符合要求",
            suggestion="请修改字体",
            expected_value=f"{expected_font} {expected_size}pt",  # 期望值
            actual_value=f"{actual_font} {actual_size}pt"        # 实际值
        ))
```

**显示效果**：
```
位置：第2页 第5段
问题：字体不符合要求
期望：宋体 12pt
实际：黑体 14pt
建议：请修改字体
```

**应用场景**：
- 字体检查：期望"宋体 12pt"，实际"黑体 14pt"
- 行间距检查：期望"1.5倍"，实际"1.0倍"
- 页边距检查：期望"25mm"，实际"20mm"
- 字号检查：期望"小四号"，实际"五号"

### 3. 多问题统计方式

**问题**：如果有5个图标有问题，显示5个问题还是1个问题？

**解决方案**：
- **检查项数量**：1个检查项未通过
- **问题数量**：5个具体问题
- **显示方式**："发现 5 个问题"

**实现方式**：
```python
class UnitLegalChecker(BaseChecker):
    def check(self, document_processor):
        issues = []
        found_errors = []  # 收集所有错误
        
        # 检查每个段落
        for para in paragraphs:
            if 'KM' in text:
                found_errors.append({
                    'page': page,
                    'wrong': 'KM',
                    'correct': 'km'
                })
        
        # 关键：为每个错误创建独立的issue
        for error in found_errors:
            issues.append(self.create_issue(
                position=f"第{error['page']}页",
                description=f"{error['wrong']} 应为 {error['correct']}"
            ))
        
        # 结果：
        # - passed = False（1个检查项未通过）
        # - len(issues) = 5（5个具体问题）
        return self.create_result(passed=False, issues=issues)
```

**统计效果**：
```

状态：✗ 未通过
问题数量：5个

问题列表：
1. 第2页 第3段：KM 应为 km
2. 第5页 第7段：Kg 应为 kg
3. 第8页 第2段：MM 应为 mm
4. 第10页 第5段：S 应为 s
5. 第12页 第1段：T 应为 t
```

### 检查结果统计

| 统计项 | 说明 | 示例 |
|--------|------|------|
| 总检查项 | 所有启用的规则数量 | 43项 |
| 通过项 | passed=True的检查项 | 36项 |
| 未通过项 | passed=False的检查项 | 5项 |
| 不适用项 | status=NOT_APPLICABLE | 2项 |
| **总问题数** | 所有issues的总和 | 15个 |

### 通过率计算

```python
# 不包括"不适用"的检查项
通过率 = 通过项 / (总检查项 - 不适用项) × 100%

示例：
- 总检查项：43
- 不适用项：2（文档中无图、无公式）
- 通过项：36
- 未通过项：5

通过率 = 36 / (43 - 2) = 36 / 41 = 87.8%
```

## 💡 使用建议

### 1. 理解"不适用"状态
- "不适用" ≠ "通过"
- "不适用" = 文档中没有相关内容，无需检查
- 统计时不计入通过率

### 2. 查看实际值
- 在问题详情中查看期望值和实际值
- 了解具体哪里不符合要求
- 根据实际值调整文档

### 3. 问题统计
- 1个检查项未通过可能包含多个问题
- 问题数量 = issues列表长度
- 每个问题都有独立的位置和建议
